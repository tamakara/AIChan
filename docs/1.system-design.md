# 1号文档：系统设计

## 1.1 设计目标

本版本目标：

1. 从静态插件模式切换为 MCPHub 动态工具模式；
2. 保留核心信号机制（SignalHub + 串行处理）；
3. 通过通道轮询触发实现消息闭环。

## 1.2 关键设计原则

1. **接入与触发解耦**
   - MCPHub 只负责工具连接与包装；
   - 通道触发由独立轮询组件处理。
2. **单核顺序推理**
   - 所有触发最终汇入 `SignalHub` 队列；
   - `SignalProcessor` 串行处理，保证上下文顺序稳定。
3. **动态工具快照**
   - 每次处理信号前从 MCPHub 读取最新工具快照；
   - 按快照重建 Agent，避免热更新复杂状态。

## 1.3 MCPHub 工具模型

- MCPHub 通过 `MCP_SERVER_URLS` 连接一个或多个 MCP Server；
- `session.list_tools()` 动态发现 `mcp.types.Tool`；
- 统一包装为 `StructuredTool`，命名规则：`server_alias__tool_name`；
- 工具执行统一转发到 `session.call_tool(name, arguments)`。

## 1.4 信号流转路径

1. `ChannelPollTrigger` 轮询各通道 `GET /v1/messages?after_id=*`；
2. 发现新增 `user` 消息后触发 `AgentSignal`；
3. `SignalHub` 将信号入队并串行消费；
4. `SignalProcessor` 拉取消息 -> 从 MCPHub 获取工具快照 -> 推理 -> 回写 assistant。
