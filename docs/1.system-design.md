# 1号文档：系统设计

## 1.1 设计目标

本版本目标：

1. 从静态插件模式切换为动态注册中心模式；
2. 保留核心信号机制（SignalHub + 串行处理）；
3. 通过网关动态注册实现可插拔工具与通道接入。

## 1.2 关键设计原则

1. **接入与触发解耦**
   - 注册中心只负责接入与映射；
   - 信号触发由独立组件处理。
2. **单核顺序推理**
   - 所有触发最终汇入 `SignalHub` 队列；
   - `SignalProcessor` 串行处理，保证上下文顺序稳定。
3. **动态工具快照**
   - 每次处理信号前读取最新工具快照；
   - 按快照重建 Agent，避免热更新复杂状态。

## 1.3 注册中心模型

网关注册请求体（当前版本）：

- `name`
- `type`：`channel` 或 `tool`
- `base_url`
- `openapi_path`
- `sse_path`（仅 `channel` 必填）

严格约束：

- `channel` 必须提供 `sse_path`；
- `tool` 禁止提供 `sse_path`。

## 1.4 信号流转路径

1. 网关 SSE 事件被 `registry` 监听并写入 `global_event_bus`；
2. `RegistrySignalTrigger` 消费总线，仅在 `user` 新消息时触发 `AgentSignal`；
3. `SignalHub` 将信号入队并串行消费；
4. `SignalProcessor` 拉取消息 -> 推理 -> 回写 assistant。
