# 0号文档：系统边界

## 0.1 大脑（AIChan Brain）边界

大脑进程负责三件事：

1. 管理通道配置并维护信号处理流程；
2. 通过 MCPHub 管理 MCP Server 并映射为动态工具集合；
3. 消费信号并驱动 Agent 推理回写。

大脑不负责：

- 网关本地消息存储实现；
- 网关接入协议的业务细节（仅按统一契约调用）。

## 0.2 网关（Gateway）边界

网关进程负责两件事：

1. 对外提供消息 API（`/v1/messages`）与事件流（`/v1/events`）；
2. 作为被动外设，由大脑按约定地址轮询与调用。

网关不负责：

- 直接触发大脑推理；
- 维护大脑侧信号队列状态。

## 0.3 信号机制边界

- `mcp_hub`：只做 MCP 连接管理、工具发现与工具包装；
- `ChannelPollTrigger`：只做“消息轮询 -> AgentSignal”触发；
- `SignalHub`：只做排队与顺序消费；
- `SignalProcessor`：只做消息拉取、推理和回写。
